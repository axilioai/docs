Every request to Axilio is authenticated with an **API key**. This page shows how
to create one, give it to the SDK, and use it directly over HTTP.

## Create an API key

Generate a key from the [dashboard](https://axilio.ai/dashboard) under
**Settings → API keys**. Keys are prefixed `axl_`.

<Warning>
  An API key is shown **once**, at creation. Store it somewhere safe — a secrets
  manager or an environment variable — and never commit it to source control.
</Warning>

Each key is scoped to a **single organization**. If you belong to more than one,
create a separate key per organization and use the one for the account you want to
bill and operate against.

## Configure the SDK

The client reads your key from the `AXILIO_API_KEY` environment variable:

```bash
export AXILIO_API_KEY=axl_your_key_here
```

```python
from axilio.platform import Client

client = Client()  # reads AXILIO_API_KEY
```

Or pass it explicitly:

```python
from axilio.platform import Client

client = Client(api_key="axl_your_key_here")
```

## Configuration options

`Client()` accepts a few options, each with an environment-variable fallback:

| Option | Environment variable | Default | Description |
|---|---|---|---|
| `api_key` | `AXILIO_API_KEY` | — | Your API key (`axl_…`). Required. |
| `base_url` | `AXILIO_BASE_URL` | `https://api.axilio.ai` | The API host. |
| `timeout` | — | `30.0` | Per-request timeout, in seconds. |
| `max_retries` | — | `3` | Retries on `429`/`5xx`/network errors. |

## Calling the API directly

The SDK is the recommended path, but the REST API is plain HTTP. Send your key in
the `X-Axilio-Api-Key` header against the base URL `https://api.axilio.ai`, under
the `/api/v1` path:

```bash
curl https://api.axilio.ai/api/v1/devices/available \
  -H "X-Axilio-Api-Key: axl_your_key_here"
```

<Note>
  `X-Axilio-Api-Key` is for API keys. The dashboard uses a separate
  `Authorization: Bearer` session token — it is not interchangeable with an API
  key.
</Note>
