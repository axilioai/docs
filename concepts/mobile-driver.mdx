`MobileDriver` is your primary interface for automating a device. It connects to the on-device agent inside a sandbox, then exposes a chainable selector surface that lets you find elements by text, describe them in natural language, or capture the full screen as a typed snapshot. Every selector returns an `Element` you can immediately act on — tap it, type into it, long-press it, or swipe from it to another element — without ever needing to track raw pixel coordinates yourself. For cases where you need precise control, the driver also exposes low-level methods that accept explicit coordinates.

## Connecting to a device

Inside a workflow run (sandbox mode), call `MobileDriver.connect()` to pair with the pre-allocated device. The driver connects over a Unix domain socket to the local on-device agent, which handles all interaction with the physical hardware.

```python
from axilio.drivers.mobile import MobileDriver

driver = MobileDriver.connect()
```

Use the driver as a context manager to ensure the underlying transport is released cleanly when your script finishes:

```python
with MobileDriver.connect() as driver:
    # All automation happens here.
    ...
```

<Note>
  `MobileDriver.connect()` is designed to be called inside a workflow script running in sandbox mode. Outside of a sandbox you will need to provide a pre-allocated `DeviceHandle` — see the [Devices](/concepts/devices) concept page for details.
</Note>

## The three selector strategies

Axilio provides three complementary ways to locate elements on screen. Choose the right one for each situation to balance speed, accuracy, and flexibility.

<CardGroup cols={3}>
  <Card title="find_text()" icon="text">
    OCR-based. Fast and deterministic. Use this when you know the exact string that appears on screen.
  </Card>
  <Card title="find()" icon="magnifying-glass">
    VLM semantic search. Use this when you want to describe what to find in plain English instead of matching text verbatim.
  </Card>
  <Card title="observe()" icon="eye">
    Full-screen snapshot. Returns a `Screen` object containing all detected text and icons. Use this when you need to inspect multiple elements at once.
  </Card>
</CardGroup>

### `find_text()` — OCR-based text search

`find_text()` runs an OCR pass on the current screen and returns the first `Element` whose text matches your input. By default the match is case-insensitive and substring-based. Pass `exact=True` to require a full string match.

```python
# Finds any element containing "Sign in" (case-insensitive)
el = driver.find_text("Sign in")
if el:
    el.tap()

# Exact match only
el = driver.find_text("Sign in", exact=True)
```

`find_text()` returns `None` when no match is found, so always check the result before calling actions, or use `wait_for_text()` to poll until it appears.

### `find()` — VLM semantic search

`find()` sends the current screen to a vision-language model and asks it to locate whatever your `query` describes. Use this for icons, images, unlabelled buttons, or any UI element that doesn't have a reliable text label.

```python
# Find the avatar image in the top-right corner
avatar = driver.find(query="profile avatar in the top-right corner")
avatar.tap()

# Find an icon by its visual meaning
back_btn = driver.find(query="back navigation arrow")
back_btn.tap()
```

<Info>
  `find()` raises `ElementNotFoundError` (not `None`) when the VLM cannot locate a matching element. Wrap it in a `try / except` or use `wait_for()` with a predicate when the element may not be present immediately.
</Info>

### `observe()` — full-screen snapshot

`observe()` captures the current frame and returns a `Screen` containing all detected text elements and icons. Use it when you need to inspect several parts of the screen in a single round trip, or when you want to pass the raw snapshot to your own processing logic.

```python
screen = driver.observe()

# Check a specific piece of text
if screen.find_text("Welcome back"):
    print("Logged in successfully")

# Iterate over all on-screen text
for el in screen.texts:
    print(el.text, el.confidence)

# Inspect detected icons
print(f"Found {len(screen.icons)} icons")
```

## OCR engines

Both `find_text()`, `find_all_text()`, and `observe()` accept an `ocr_engine` parameter that controls which OCR model processes the frame.

| Engine | Default | Accuracy | Billing |
|---|---|---|---|
| `"free"` | ✓ | Good — powered by RapidOCR | Included in device time |
| `"premium"` | — | Higher — larger model | Billed per inference |

```python
# Use the premium engine for small or stylised text
el = driver.find_text("©2024", ocr_engine="premium")
```

<Tip>
  Start with `"free"` (the default) and only switch to `"premium"` if OCR accuracy becomes a problem — for example, small fonts, decorative typography, or text on complex backgrounds.
</Tip>

## The `Element` type

Every selector returns one or more `Element` objects. An `Element` is an immutable, frozen dataclass that carries the location of the detected UI element and provides chainable action methods.

<ResponseField name="bbox" type="BBox">
  The bounding box of the element in frame-space pixels: `{"x": int, "y": int, "width": int, "height": int}`. The origin is the top-left corner of the screen.
</ResponseField>

<ResponseField name="center" type="Coords">
  The computed centre point of the bounding box: `{"x": int, "y": int}`. All element-level tap and press actions use this point.
</ResponseField>

<ResponseField name="confidence" type="float">
  A 0–1 score representing how confident the underlying model was in this detection. Higher is better.
</ResponseField>

<ResponseField name="text" type="str | None">
  The recognized text content, for OCR-sourced elements. `None` for purely VLM-located elements that were found by visual appearance rather than text.
</ResponseField>

<ResponseField name="source" type='"ocr" | "vlm"'>
  Which model produced this element — `"ocr"` for text-based results and `"vlm"` for vision-language model results.
</ResponseField>

### Element actions

All actions are chainable directly off the returned `Element`:

```python
# Tap the centre of the element
driver.find_text("Continue").tap()

# Long-press for 1.2 seconds
driver.find_text("Hold to confirm").long_press(duration_ms=1200)

# Tap to focus, then type text
driver.find_text("Search").type_into("axilio automation")

# Swipe from one element to another
top = driver.find_text("Top item")
bottom = driver.find_text("Bottom item")
top.swipe_to(bottom, duration_ms=400)
```

| Method | Description |
|---|---|
| `el.tap()` | Tap once at the element's centre |
| `el.long_press(duration_ms=800)` | Press and hold for the given duration |
| `el.type_into(text)` | Tap the element to focus it, then type |
| `el.swipe_to(other, duration_ms=300)` | Swipe from this element's centre to another's |

## The `Screen` type

`observe()` returns a `Screen` — an immutable snapshot of what was on the display at the moment of capture.

<ResponseField name="texts" type="list[Element]">
  All OCR-detected text elements in the frame, in order of detection.
</ResponseField>

<ResponseField name="icons" type="list[IconBox]">
  All icon detections in the frame. Each `IconBox` has a `bbox`, `center`, and `confidence`, but no text label — the icon model detects presence and location only.
</ResponseField>

<ResponseField name="hash" type="str">
  A perceptual hash of the frame. Compare hashes between two `observe()` calls to check whether the screen changed.
</ResponseField>

<ResponseField name="width" type="int">
  Frame width in pixels.
</ResponseField>

<ResponseField name="height" type="int">
  Frame height in pixels.
</ResponseField>

<ResponseField name="captured_at" type="datetime">
  UTC timestamp of when the frame was captured by the on-device agent.
</ResponseField>

`Screen` also has two convenience search methods:

```python
screen = driver.observe()

# Find the first element matching a substring
el = screen.find_text("Inbox")

# Find all elements matching a substring
els = screen.find_all_text(contains="notification")

# Find all elements matching a regex
els = screen.find_all_text(pattern=r"\d+ unread")
```

## The `IconBox` type

Icon detections are returned as `IconBox` objects in `screen.icons`. The icon detection model uses a single class for all icons — it tells you *where* icons are, not *what* they are.

<ResponseField name="bbox" type="BBox">
  Bounding box of the detected icon in frame-space pixels.
</ResponseField>

<ResponseField name="center" type="Coords">
  Pre-computed centre point — pass directly to `driver.tap()`.
</ResponseField>

<ResponseField name="confidence" type="float">
  Detection confidence from the underlying YOLO model (0–1).
</ResponseField>

```python
screen = driver.observe()
for icon in screen.icons:
    print(icon.center, icon.confidence)

# Tap the first detected icon
if screen.icons:
    driver.tap(screen.icons[0].center)
```

## Wait helpers

Polling manually with `time.sleep()` is fragile and slow. Axilio provides three built-in wait helpers that poll at a configurable interval and raise `TimeoutError` when the deadline passes.

### `wait_for_text()` — wait until text appears

```python
# Wait up to 15 seconds for a loading screen to clear
el = driver.wait_for_text("Home", timeout=15.0, poll_ms=500)
el.tap()
```

### `wait_until_gone()` — wait until text disappears

```python
# Wait for a spinner to disappear before proceeding
driver.wait_until_gone("Loading...", timeout=20.0)
```

### `wait_for()` — wait for an arbitrary predicate

Pass a callable that receives a `Screen` and returns a truthy value when the condition is met. This is the most flexible option — use it to check multiple conditions simultaneously.

```python
def inbox_loaded(screen: Screen) -> bool:
    return (
        screen.find_text("Inbox") is not None
        and screen.find_text("Loading") is None
    )

screen = driver.wait_for(inbox_loaded, timeout=20.0)
```

<Warning>
  All wait helpers raise `axilio.drivers.mobile.TimeoutError` when the deadline elapses without the condition being met. Catch this explicitly if you want to handle the timeout gracefully rather than letting the script fail.
</Warning>

## Direct driver actions

In addition to element-level actions, `MobileDriver` exposes direct methods that accept raw coordinates or driver-level inputs. Use these when element selectors are not appropriate.

<CodeGroup>
```python Tap coordinates
# Tap at an absolute pixel coordinate
driver.tap({"x": 540, "y": 960})
```

```python Swipe
# Swipe from top of screen to bottom (scroll down)
driver.swipe(
    start={"x": 540, "y": 300},
    end={"x": 540, "y": 1600},
    duration_ms=400,
)
```

```python Type text
# Type text directly — no element focus required
driver.type_text("hello world")
```

```python Key press
from axilio.drivers.mobile import Key

# Press the hardware Back button
driver.key_press(Key.BACK)

# Return to the Home screen
driver.key_press(Key.HOME)

# Open the Recents / App Switcher
driver.key_press(Key.RECENTS)
```

```python Screenshot
# Capture the screen as PNG bytes
png_bytes = driver.screenshot()
with open("screen.png", "wb") as f:
    f.write(png_bytes)
```
</CodeGroup>

## The `Key` class

`Key` provides constants for common hardware buttons, mapped to HID consumer-page key codes. Import it from `axilio.drivers.mobile`.

```python
from axilio.drivers.mobile import Key

driver.key_press(Key.HOME)        # Home button
driver.key_press(Key.BACK)        # Back button
driver.key_press(Key.MENU)        # Menu button
driver.key_press(Key.RECENTS)     # Recents / App Switcher
driver.key_press(Key.POWER)       # Power button
driver.key_press(Key.VOLUME_UP)   # Volume up
driver.key_press(Key.VOLUME_DOWN) # Volume down
driver.key_press(Key.MUTE)        # Mute
```

## Complete example

The following example shows a typical login automation that combines multiple selector strategies, wait helpers, and direct actions:

```python
from axilio.drivers.mobile import MobileDriver, Key
import axilio

client = axilio.Client()

with client.mobile.session(apps=["com.example.myapp"]) as handle:
    with MobileDriver.connect() as driver:
        # Wait for the app's splash screen to clear
        driver.wait_until_gone("Loading", timeout=15.0)

        # Tap the email field and type credentials
        driver.find_text("Email").type_into("user@example.com")
        driver.find_text("Password").type_into("s3cr3t!")

        # Tap the Sign In button
        driver.find_text("Sign In").tap()

        # Wait until the dashboard is ready
        driver.wait_for_text("Dashboard", timeout=20.0)

        # Take a screenshot to confirm
        png = driver.screenshot()
        print(f"Screenshot captured: {len(png)} bytes")
```
