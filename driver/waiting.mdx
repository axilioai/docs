title: "Wait for the screen to settle"
sidebarTitle: "Waiting"

---

Real apps load, animate, and transition. Acting before the UI is ready is the most
common cause of flaky automation. Instead of guessing with `sleep()`, **wait for a
condition** — the driver polls the device until it's true.

## Wait for text to appear

`wait_for_text` polls until the text shows up, then returns the
[element](/driver/actions) so you can act on it in one line. It raises
`TimeoutError` if the text never appears:

```python
from axilio.platform import Client

client = Client()

with client.session("ANDROID") as driver:
    driver.find_text("Sign in").tap()
    driver.wait_for_text("Welcome").tap()   # blocks until "Welcome" appears
```

## Wait for text to disappear

`wait_until_gone` is the inverse — it blocks until the text is no longer on screen.
Use it to wait out spinners and loading states:

```python
driver.wait_until_gone("Loading")
```

## Wait for any condition

`wait_for` polls [`observe()`](/driver/read-the-screen) and returns the `Screen` as
soon as your predicate is truthy. Use it for conditions that aren't a single piece
of text:

```python
# Wait until at least 10 list rows have rendered.
screen = driver.wait_for(lambda s: len(s.texts) >= 10)
```

## Tuning the wait

All three accept the same controls:

| Parameter | Default | Description |
|---|---|---|
| `timeout` | `10.0` | Seconds to keep polling before raising `TimeoutError`. |
| `poll_ms` | `300` | How often to re-check, in milliseconds. |

```python
driver.wait_for_text("Confirmed", timeout=30).tap()
```

<Warning>
  Every wait raises `TimeoutError` (from `axilio.drivers.mobile`) when the condition
  isn't met in time. Catch it to handle screens that may not appear:

  ```python
  from axilio.drivers.mobile import TimeoutError

  try:
      driver.wait_for_text("Approved", timeout=15).tap()
  except TimeoutError:
      ...  # take a different path
  ```
</Warning>

<Tip>
  Prefer waits over `time.sleep()`. A wait returns the instant the condition is
  met — usually faster than a fixed sleep, and far more reliable.
</Tip>
