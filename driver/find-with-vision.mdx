When there's no text to match — an icon, an image, a button you can only describe
— tell Axilio what you want in plain English. That's `find`.

```python
from axilio.platform import Client

client = Client()

with client.session("ANDROID") as driver:
    driver.find(query="the heart icon under the photo").tap()
```

## How it's different from find_text

- **It matches what you describe**, not exact text — so it works on things with no
  readable label.
- **It speaks up when it can't find something.** [`find_text`](/driver/find-by-text)
  just comes back empty; `find` raises an error you can catch:

```python
from axilio.drivers.mobile import ElementNotFoundError

try:
    driver.find(query="the shopping cart").tap()
except ElementNotFoundError:
    print("no cart on this screen")
```

## Options

```python
driver.find(query="the blue Continue button", timeout=10)
```

- **`timeout`** — how long to keep looking before giving up (seconds). It keeps
  checking the screen, so this also waits for the thing to appear.

## When to use it

- **Describe it** when there's no text to match, or it's easier to describe by how
  it looks.
- **Use [text](/driver/find-by-text)** when there's a clear label — it's faster.

<Note>
  Describing something is powered by AI, so it costs a little more than finding by
  text. On steps you run a lot, prefer text.
</Note>
