When the thing you want has no usable text — an icon, an image, "the blue button
next to the avatar" — describe it in plain language and let a vision model find it.
That's `find(query=...)`.

```python
from axilio.platform import Client

client = Client()

with client.session("ANDROID") as driver:
    driver.find(query="the heart icon under the photo").tap()
```

## How it differs from text

`find` returns a single [`Element`](/driver/actions), just like
[`find_text`](/driver/find-by-text) — but with two important differences:

- **It describes intent, not a label.** The query is matched visually, so it works
  on elements that have no readable text.
- **It raises on a miss.** Where `find_text` returns `None`, `find` raises
  `ElementNotFoundError` if it can't locate the target — so a failed selector
  surfaces loudly instead of silently returning nothing.

```python
from axilio.drivers.mobile import ElementNotFoundError

try:
    driver.find(query="the shopping cart icon").tap()
except ElementNotFoundError:
    print("cart not visible on this screen")
```

## Options

```python
el = driver.find(
    query="the blue Continue button",
    timeout=10.0,   # seconds to keep looking before raising
    model=None,     # pin a specific vision model; defaults to the recommended one
)
```

- **`timeout`** — how long to keep looking before raising `ElementNotFoundError`.
  `find` polls the screen, so this doubles as a wait for the element to appear.
- **`model`** — pin a specific vision model. Leave it unset to use the recommended
  default.

## When to use vision

- **Reach for vision** when there's no text to match, when the label is unreliable,
  or when the target is best described by its appearance or position.
- **Prefer [text](/driver/find-by-text)** when a stable visible label exists — it's
  faster, deterministic, and cheaper.

<Note>
  Vision calls are billed separately from device time. Favor
  [text selectors](/driver/find-by-text) on hot paths, and use vision where it
  earns its keep.
</Note>
