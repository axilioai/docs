The `MobileDriver` is the control surface for a device. A
[session](/quickstart) hands you one; from there you **find** things on screen and
**act** on them.

```python
from axilio.platform import Client

client = Client()

with client.session("ANDROID") as driver:
    driver.find_text("Settings").tap()
```

## Two ways to find, three ways to act

The driver gives you two kinds of selector and a set of actions:

- **Text selectors** — [`find_text`](/driver/find-by-text) locates on-screen text
  with fast, on-device OCR. Use it whenever the thing you want has a visible label.
- **Vision selectors** — [`find(query=...)`](/driver/find-with-vision) describes an
  element in natural language and locates it with a vision model. Use it for icons,
  images, and anything OCR can't name.
- **Actions** — every selector returns an [`Element`](/driver/actions) you act on:
  `tap()`, `type_into()`, `long_press()`, `swipe_to()`. You can also send
  [raw input](/driver/actions) and [hardware keys](/driver/hardware-keys) directly.

## The Element you get back

Selectors return an `Element` — a located thing on screen — that you act on
directly and inspect:

| Field | Description |
|---|---|
| `center` | The `{"x", "y"}` tap point at the element's center. |
| `bbox` | The bounding box: `{"x", "y", "width", "height"}`. |
| `text` | The recognized text, if any. |
| `confidence` | How sure the locator is, from `0` to `1`. |
| `source` | `"ocr"` for a text match, `"vlm"` for a vision match. |

```python
el = driver.find_text("Sign in")
print(el.text, el.center, el.confidence)
el.tap()
```

<Warning>
  `find_text` returns `None` when nothing matches, so calling `.tap()` straight
  away can raise `AttributeError` if the screen hasn't loaded. Guard the result,
  or use [`wait_for_text`](/driver/waiting) to block until it appears.
  `find(query=...)` instead raises `ElementNotFoundError` on a miss.
</Warning>
