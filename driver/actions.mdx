Once you've [found an element](/driver/overview), you act on it. Actions come in
two forms: **chained off an element**, or sent to **raw coordinates** when you
already know where to act.

## Act on an element

Every selector returns an `Element` whose actions you call directly:

```python
from axilio.platform import Client

client = Client()

with client.session("ANDROID") as driver:
    driver.find_text("Settings").tap()
    driver.find_text("Search").type_into("axilio")
    driver.find_text("Item").long_press()
    driver.find_text("Card").swipe_to(driver.find_text("Archive"))
```

| Method | Description |
|---|---|
| `tap()` | Tap the element's center. |
| `type_into(text)` | Tap the element, then type `text` into it. |
| `long_press(duration_ms=800)` | Press and hold the element. |
| `swipe_to(other)` | Swipe from this element to another element. |

<Tip>
  These act at the element's `center`. To act somewhere else relative to an
  element, read its `bbox` and use the raw input below.
</Tip>

## Act on raw coordinates

When you already know the pixel location — from a `bbox`, a prior screenshot, or a
fixed layout — drive the device directly. Coordinates are `{"x": int, "y": int}`:

```python
driver.tap({"x": 540, "y": 1180})
driver.long_press({"x": 540, "y": 1180}, duration_ms=1000)
driver.swipe({"x": 540, "y": 1600}, {"x": 540, "y": 400})  # scroll up
driver.type_text("hello")  # type into whatever currently has focus
```

| Method | Description |
|---|---|
| `tap(coords)` | Tap once at `coords`. |
| `long_press(coords, duration_ms=800)` | Press and hold at `coords`. |
| `swipe(start, end, duration_ms=300)` | Swipe from `start` to `end`. |
| `type_text(text)` | Type into the currently focused field. |

<Note>
  `swipe` is how you scroll: swipe **up** (end `y` lower than start `y`) to move
  *down* a list, and vice versa. Increase `duration_ms` for a slower, shorter
  scroll.
</Note>
