Instead of looking for one thing at a time, `observe` reads **everything on the
screen at once** — all the text and icons — so you can look through it in your
code.

```python
from axilio.platform import Client

client = Client()

with client.session("ANDROID") as driver:
    screen = driver.observe()
    print(len(screen.texts), "pieces of text")
```

## What you get

`observe` hands back a snapshot of the screen, with:

- **`texts`** — every piece of text it found
- **`icons`** — every icon it found
- **`width` and `height`** — the screen size, in pixels

You can search the snapshot without going back to the phone:

```python
screen = driver.observe()

sign_in = screen.find_text("Sign in")
prices = screen.find_all_text(contains="$")
```

This is the difference from [`driver.find_text`](/driver/find-by-text): the driver
looks at the phone fresh every time, while searching a snapshot reuses the one you
already took. Use `observe` when you want to make several decisions about the same
screen.

## A picture instead?

`observe` gives you the text and icons. If you want the screen as an **image** — to
save, log, or look at — use [`screenshot`](/driver/screenshots).
