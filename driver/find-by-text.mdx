`find_text` finds something on the screen by the text you can see on it — a
button, a menu item, a label. It's the fastest, most reliable way to find things.

```python
from axilio.platform import Client

client = Client()

with client.session("ANDROID") as driver:
    driver.find_text("Settings").tap()
```

## Find one thing

`find_text` hands back what it found, or nothing if the text isn't there:

```python
result = driver.find_text("Sign in")
if result:
    result.tap()
```

<Warning>
  Because it can come back empty, calling `.tap()` right away can fail if the
  screen hasn't loaded yet. To be safe, [wait for the text](/driver/waiting)
  first — this waits until it appears, then taps it:

  ```python
  driver.wait_for_text("Sign in").tap()
  ```
</Warning>

Want an exact match instead of a partial one? Pass `exact=True`:

```python
driver.find_text("OK", exact=True).tap()
```

## Find every match

`find_all_text` hands back a list of everything that matches — handy for lists and
feeds. Filter by what the text contains:

```python
for item in driver.find_all_text(contains="Notification"):
    print(item.text)
```

## Text, or plain English?

- **Use text** when the thing has a label you can read. It's faster and exact.
- **Use [plain English](/driver/find-with-vision)** when it doesn't — an icon, an
  image, or something easier to describe than to name.
