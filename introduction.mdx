Axilio is a cloud-hosted mobile automation platform that puts real Android handsets — connected to live carrier networks — under programmatic control. Unlike emulators or screenshot-replay tools, your code talks to physical hardware: calls connect, push notifications arrive on schedule, carrier-gated flows behave exactly as they would for a real user. You request a device through the SDK, drive it with a clean chainable API, and release it when you're done. Axilio handles provisioning, networking, and teardown so you can focus entirely on the automation logic.

## Why Axilio

Most mobile automation runs inside emulators that diverge from production hardware in ways that matter — missing sensors, spoofed network stacks, and artificial latency. Axilio eliminates that class of problem entirely.

## Core SDK concepts

The Python SDK ships two primary entry points that work together: a platform client that manages device lifecycle and account resources, and a driver that controls the device you've acquired.

### `axilio.platform.Client`

`Client` is your connection to the Axilio API. Construct it once per process — it holds the HTTP connection pool and your credentials — and share it across calls. Every resource in the platform is accessible as an attribute:

```python
from axilio.platform import Client

client = Client()  # reads AXILIO_API_KEY from the environment
```

Through `Client` you can allocate devices (`client.mobile`), manage workflows and runs (`client.workflows`, `client.runs`), inspect usage and billing (`client.usage`, `client.billing`), and administer API keys and organisation members (`client.api_keys`, `client.org`).

### `axilio.drivers.MobileDriver`

`MobileDriver` wraps a `DeviceHandle` returned by `client.mobile.allocate()` (or `client.mobile.session()`) and exposes a selector-based control surface for the live device. Every selector returns an `Element` whose action methods chain fluently:

```python
from axilio.drivers import MobileDriver

with client.mobile.session(platform="android") as device:
    driver = MobileDriver(device)

    driver.find_text("Sign in").tap()
    driver.find_text("Email").type_into("you@example.com")
    driver.find(query="the blue Continue button").tap()
    png = driver.screenshot()
```

`find_text` uses fast on-device OCR; `find` sends a natural-language query to Argus for anything that text alone can't pin down.

## Argus — vision inference

Argus is Axilio's cloud vision inference service. It runs object detection (YOLO), OCR (free and premium engines), and a VLM-backed element locator that accepts plain-English descriptions of UI elements. The `MobileDriver` calls Argus transparently when you use `driver.find(query=...)`, but you can also call it directly through `client.argus` to run inference over arbitrary images — useful for custom screenshot analysis, labelling pipelines, or pre-computing OCR results before driving a session.
