`client.mobile` gives you full control over physical device lifecycle. You can inspect which devices are currently free, pin allocations to a specific geographic rack, pre-install apps before your session starts, and release hardware when you're done. The recommended pattern is to use `session()` as a context manager so the device is always returned to the pool — even if your script raises an exception. When you need more control over timing, call `allocate()` and `deallocate()` directly, or call `DeviceHandle.deallocate()` on the handle itself.

## `session()`

```python
client.mobile.session(
    *,
    platform: Platform = "android",
    location: str | None = None,
    apps: list[str] | None = None,
    timeout: float = 60.0,
) -> Iterator[DeviceHandle]
```

An all-in-one context manager that allocates a device, yields the `DeviceHandle`, then deallocates automatically when the `with` block exits — whether it exits normally or due to an exception. This is the recommended way to use devices in scripts and CI pipelines.

<ParamField body="platform" type="'android' | 'iphone'" default='"android"'>
  The device operating system to request.
</ParamField>

<ParamField body="location" type="str | None" default="None">
  ID of the rack location to pin the device to (e.g. `"us-east-1"`). Omit to
  let Axilio choose the nearest available device.
</ParamField>

<ParamField body="apps" type="list[str] | None" default="None">
  Bundle IDs of apps to pre-install before the session starts
  (e.g. `["com.example.myapp"]`). Use `supported_apps()` to list valid IDs.
</ParamField>

<ParamField body="timeout" type="float" default="60.0">
  Seconds to wait for a matching device to become available before raising a
  `TimeoutError`.
</ParamField>

**Returns** `DeviceHandle` (via `Iterator`) — see [DeviceHandle](#devicehandle) below.

```python
from axilio.platform import Client
from axilio.drivers import MobileDriver

with Client() as client:
    with client.mobile.session(platform="android", apps=["com.example.app"]) as device:
        driver = MobileDriver.connect()
        screen = driver.observe()
        print(screen.width, screen.height)
# Device is automatically released here
```

## `allocate()`

```python
client.mobile.allocate(
    *,
    platform: Platform = "android",
    location: str | None = None,
    apps: list[str] | None = None,
    timeout: float = 60.0,
) -> DeviceHandle
```

Acquires a device and returns a `DeviceHandle` immediately. You are responsible for calling `deallocate()` or using the handle as a context manager. Device billing begins as soon as `allocate()` returns successfully.

<ParamField body="platform" type="'android' | 'iphone'" default='"android"'>
  The device operating system to request.
</ParamField>

<ParamField body="location" type="str | None" default="None">
  Rack location ID to pin the device to. Omit for automatic placement.
</ParamField>

<ParamField body="apps" type="list[str] | None" default="None">
  Bundle IDs of apps to pre-install before yielding the handle.
</ParamField>

<ParamField body="timeout" type="float" default="60.0">
  Seconds to wait for a free device before raising a `TimeoutError`.
</ParamField>

**Returns** [`DeviceHandle`](#devicehandle).

```python
with Client() as client:
    device = client.mobile.allocate(platform="iphone", location="us-west-2")
    try:
        # use device ...
        pass
    finally:
        device.deallocate()
```

## `deallocate()`

```python
client.mobile.deallocate(allocation_id: str) -> None
```

Releases the allocation identified by `allocation_id`, ending billing for that device. This is a no-op in sandbox mode. Prefer calling `DeviceHandle.deallocate()` on the handle directly, or use `session()` to have it called automatically.

<ParamField body="allocation_id" type="str" required>
  The `id` field from the `DeviceHandle` returned by `allocate()`.
</ParamField>

**Returns** `None`.

```python
with Client() as client:
    device = client.mobile.allocate(platform="android")
    # ... do work ...
    client.mobile.deallocate(device.id)
```

## `available()`

```python
client.mobile.available(
    *,
    platform: Platform | None = None,
    location: str | None = None,
    apps: list[str] | None = None,
) -> AvailableDevices
```

Returns the set of devices currently free and matching all supplied filters. Use this to check capacity before calling `allocate()`, or to build dashboards that reflect fleet availability.

<ParamField body="platform" type="'android' | 'iphone' | None" default="None">
  Filter to a specific platform. Omit to include both.
</ParamField>

<ParamField body="location" type="str | None" default="None">
  Filter to a specific rack location ID.
</ParamField>

<ParamField body="apps" type="list[str] | None" default="None">
  Only include devices that have all of these bundle IDs available.
</ParamField>

**Returns** [`AvailableDevices`](#availabledevices).

```python
with Client() as client:
    result = client.mobile.available(platform="android", location="us-east-1")
    print(f"{len(result.devices)} free out of {result.total_in_fleet} total")
```

## `locations()`

```python
client.mobile.locations() -> list[Location]
```

Returns every rack and data-centre location that Axilio operates. Pass a location `id` to `allocate()` or `session()` to pin a device to a specific geography, which is useful for latency-sensitive testing or compliance requirements.

**Returns** `list[Location]` — see [Location](#location).

```python
with Client() as client:
    for loc in client.mobile.locations():
        print(loc.id, loc.name, loc.region)
```

## `supported_apps()`

```python
client.mobile.supported_apps(
    *,
    platform: Platform | None = None,
    category: str | None = None,
) -> list[App]
```

Returns apps that can be passed in the `apps` parameter of `allocate()` and `session()`. Filter by platform or category to narrow results.

<ParamField body="platform" type="'android' | 'iphone' | None" default="None">
  Restrict to apps available on this platform.
</ParamField>

<ParamField body="category" type="str | None" default="None">
  Restrict to a named category such as `"social"`, `"finance"`, or `"browser"`.
</ParamField>

**Returns** `list[App]` — see [App](#app).

```python
with Client() as client:
    apps = client.mobile.supported_apps(platform="android", category="social")
    for app in apps:
        print(app.bundle_id, app.name)
```

## Response types

### `DeviceHandle`

An active device allocation. All fields are read-only. You can use a `DeviceHandle` directly as a context manager, which calls `deallocate()` on exit.

<ResponseField name="id" type="str">
  Allocation ID — pass this to `client.mobile.deallocate()` or use it to
  correlate billing records.
</ResponseField>

<ResponseField name="device_id" type="str">
  Stable identifier for the physical device itself (persists across
  allocations).
</ResponseField>

<ResponseField name="device_name" type="str">
  Human-readable device name, e.g. `"Pixel 7 Pro"` or `"iPhone 15"`.
</ResponseField>

<ResponseField name="platform" type="'android' | 'iphone'">
  The operating system running on this device.
</ResponseField>

<ResponseField name="location" type="str">
  The rack location ID where the device resides.
</ResponseField>

<ResponseField name="expires_at" type="datetime">
  UTC timestamp after which the allocation expires automatically if not
  refreshed or released. Poll this to avoid unexpected disconnections on long
  test runs.
</ResponseField>

<ResponseField name="signaling_url" type="str">
  WebSocket endpoint used by the SDK to establish a control channel to the
  on-device agent. You do not normally need this directly.
</ResponseField>

<ResponseField name="signaling_token" type="str">
  Short-lived bearer token that authenticates the signaling connection. Treat
  this as a secret.
</ResponseField>

<ResponseField name="ice_servers" type="list[IceServer]">
  Pre-minted STUN/TURN server credentials used by the SDK to establish the
  WebRTC control channel to the on-device agent. You do not normally need to
  use these directly.
</ResponseField>

#### `DeviceHandle.close()`

```python
device.close() -> None
```

Closes the underlying transport connection. The next method call on the same handle will reconnect automatically. Use this to free the connection when you want to pause driving without releasing the device allocation.

#### `DeviceHandle.deallocate()`

```python
device.deallocate() -> None
```

Convenience wrapper around `client.mobile.deallocate(device.id)`. Releases the device and ends billing. Calling this more than once is safe.

#### Context manager

`DeviceHandle` implements `__enter__` / `__exit__`. Use it with `with` to call `deallocate()` automatically:

```python
with Client() as client:
    with client.mobile.allocate(platform="android") as device:
        print(device.device_name)
# deallocate() called automatically on exit
```

### `AvailableDevices`

<ResponseField name="devices" type="list[AvailableDevice]">
  The devices currently free and matching your filter.

  <Expandable title="AvailableDevice fields">
    <ResponseField name="id" type="str">Stable device identifier.</ResponseField>
    <ResponseField name="name" type="str">Human-readable device name.</ResponseField>
    <ResponseField name="platform" type="'android' | 'iphone'">Device OS.</ResponseField>
    <ResponseField name="location" type="Location">Rack location object.</ResponseField>
    <ResponseField name="model" type="str">Hardware model string, e.g. `"Pixel 7 Pro"`.</ResponseField>
    <ResponseField name="os_version" type="str">OS version string, e.g. `"14.0"`.</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total_in_fleet" type="int">
  Total number of devices in the fleet matching the platform/location filter,
  including those currently allocated.
</ResponseField>

### `Location`

<ResponseField name="id" type="str">
  Short identifier used in API calls, e.g. `"us-east-1"`.
</ResponseField>

<ResponseField name="name" type="str">
  Human-readable name, e.g. `"US East (Virginia)"`.
</ResponseField>

<ResponseField name="region" type="str">
  Broad geographic region, e.g. `"us-east"`.
</ResponseField>

### `App`

<ResponseField name="bundle_id" type="str">
  The identifier to pass in the `apps` list, e.g. `"com.example.myapp"`.
</ResponseField>

<ResponseField name="name" type="str">
  Display name of the app.
</ResponseField>

<ResponseField name="platform" type="'android' | 'iphone'">
  Platform this app is available on.
</ResponseField>

<ResponseField name="category" type="str | None">
  Optional category label such as `"social"` or `"finance"`.
</ResponseField>

## Full example

```python
from axilio.platform import Client
from axilio.drivers import MobileDriver

with Client() as client:
    # Check what's available before committing
    avail = client.mobile.available(platform="android")
    print(f"{len(avail.devices)} Android devices free")

    # Run a session — device is released when the block exits
    with client.mobile.session(
        platform="android",
        apps=["com.example.myapp"],
        timeout=90.0,
    ) as device:
        print(f"Got {device.device_name} in {device.location}")
        driver = MobileDriver.connect()
        driver.find(query="Sign in button").tap()
```
