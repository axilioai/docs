`MobileDriver` is the primary interface for automating a device. You connect it to the on-device agent running inside the Axilio sandbox, then call screen-reading methods to locate elements and interaction methods to act on them. The driver exposes two selector strategies: OCR-based methods (`find_text`, `find_all_text`, `wait_for_text`) that are fast and deterministic, and a VLM-backed semantic method (`find`) that understands natural language queries and handles icons, buttons described by meaning, and layout changes. Every located element returns a chainable `Element` object that you can act on directly without tracking coordinates.

```python
from axilio.platform import Client
from axilio.drivers import MobileDriver

with Client() as client:
    with client.mobile.session(platform="android") as device:
        driver = MobileDriver.connect()

        # Semantic find using natural language
        driver.find(query="login button").tap()

        # Type credentials
        driver.find(query="email input field").type_into("user@example.com")
        driver.find(query="password input field").type_into("s3cr3t")

        driver.find(query="submit button").tap()
        driver.wait_for_text("Welcome", timeout=15.0)
```

## Connecting

### `MobileDriver.connect()`

```python
@classmethod
MobileDriver.connect(*, socket_path: str | None = None) -> MobileDriver
```

Class method that creates a `MobileDriver` connected to the sandbox's pre-allocated device via the daemon Unix socket. Call this inside a `client.mobile.session()` block after the device is allocated.

<ParamField body="socket_path" type="str | None" default="None">
  Override the Unix socket path used to reach the local daemon. When `None`
  the SDK uses the default path injected by the sandbox environment. You
  rarely need to set this explicitly.
</ParamField>

**Returns** `MobileDriver`.

`MobileDriver` also supports use as a context manager, which calls `close()` on exit:

```python
with MobileDriver.connect() as driver:
    screen = driver.observe()
```

## Screen observation

### `observe()`

```python
driver.observe(*, ocr_engine: OcrEngine = "free") -> Screen
```

Captures the current device frame and returns a fully populated `Screen` snapshot. Every OCR-based selector calls `observe()` internally, so you only need to call it directly when you want to inspect the raw screen data or call multiple selectors against the same frame.

<ParamField body="ocr_engine" type="OcrEngine" default='"free"'>
  OCR backend to use. `"free"` uses the bundled lightweight model. Pass a
  premium engine identifier for higher accuracy on complex layouts.
</ParamField>

**Returns** [`Screen`](#screen).

```python
screen = driver.observe()
print(f"{screen.width}×{screen.height}, {len(screen.texts)} text elements")
```

## Text selectors

### `find_text()`

```python
driver.find_text(
    text: str,
    *,
    exact: bool = False,
    ocr_engine: OcrEngine = "free",
) -> Element | None
```

Returns the first OCR element whose text matches `text`, or `None` if no match is found. Triggers one `observe()` call internally. Use `wait_for_text()` if the element might not appear immediately.

<ParamField body="text" type="str" required>
  The string to search for. By default the match is case-insensitive and
  substring-based.
</ParamField>

<ParamField body="exact" type="bool" default="False">
  When `True` the element's text must match `text` exactly, including case.
</ParamField>

<ParamField body="ocr_engine" type="OcrEngine" default='"free"'>
  OCR backend to use for this observation.
</ParamField>

**Returns** `Element | None`.

```python
el = driver.find_text("Sign in")
if el:
    el.tap()
```

### `find_all_text()`

```python
driver.find_all_text(
    *,
    contains: str | None = None,
    pattern: str | None = None,
    ocr_engine: OcrEngine = "free",
) -> list[Element]
```

Returns every OCR element matching the supplied criteria. Pass either `contains` or `pattern` — passing both raises `ValueError`. Omitting both returns all OCR elements on the current screen.

<ParamField body="contains" type="str | None" default="None">
  Case-insensitive substring filter. Mutually exclusive with `pattern`.
</ParamField>

<ParamField body="pattern" type="str | None" default="None">
  Regular expression applied to each element's text via `re.search()`. Mutually exclusive with `contains`.
</ParamField>

<ParamField body="ocr_engine" type="OcrEngine" default='"free"'>
  OCR backend to use.
</ParamField>

**Returns** `list[Element]`. Raises `ValueError` if both `contains` and `pattern` are supplied.

```python
# Find all price-like strings on the screen
prices = driver.find_all_text(pattern=r"\$\d+\.\d{2}")
for el in prices:
    print(el.text, el.center)
```

## Semantic selector

### `find()`

```python
driver.find(
    *,
    query: str,
    timeout: float = 10.0,
    ocr_engine: OcrEngine = "free",
    model: str | None = None,
) -> Element
```

Uses the Argus VLM to locate an element described in natural language. Unlike OCR selectors, `find()` understands intent: it can locate icons, buttons described by function rather than label, and elements whose text varies. The call is routed through the on-device agent.

<ParamField body="query" type="str" required>
  A natural language description of the element, e.g. `"the blue login button"`,
  `"search bar at the top of the screen"`, or `"X close icon"`.
</ParamField>

<ParamField body="timeout" type="float" default="10.0">
  Seconds to allow for the VLM response before raising `TimeoutError`.
</ParamField>

<ParamField body="ocr_engine" type="OcrEngine" default='"free"'>
  OCR backend used for any text extraction within the VLM pipeline.
</ParamField>

<ParamField body="model" type="str | None" default="None">
  Override the default Argus model. Omit to use the platform default.
</ParamField>

**Returns** `Element`.  
**Raises** `ElementNotFoundError` if no element on screen matches the query.

```python
el = driver.find(query="checkout button", timeout=15.0)
el.tap()
```

<Note>
  `find()` is slower than OCR-based methods because it calls the VLM on every
  invocation. Prefer `find_text()` or `wait_for_text()` for elements with
  stable, known labels, and reserve `find()` for icons and semantically
  described UI.
</Note>

## Wait methods

### `wait_for_text()`

```python
driver.wait_for_text(
    text: str,
    *,
    timeout: float = 10.0,
    poll_ms: int = 300,
    exact: bool = False,
    ocr_engine: OcrEngine = "free",
) -> Element
```

Polls `find_text()` at `poll_ms` millisecond intervals until the element appears or `timeout` seconds elapse.

<ParamField body="text" type="str" required>
  Text to wait for.
</ParamField>

<ParamField body="timeout" type="float" default="10.0">
  Total seconds to wait before raising `TimeoutError`.
</ParamField>

<ParamField body="poll_ms" type="int" default="300">
  Milliseconds between each `observe()` poll.
</ParamField>

<ParamField body="exact" type="bool" default="False">
  Whether to require an exact case-sensitive match.
</ParamField>

<ParamField body="ocr_engine" type="OcrEngine" default='"free"'>
  OCR backend to use on each poll.
</ParamField>

**Returns** `Element` — the element once it appears.  
**Raises** `TimeoutError` if the text does not appear within `timeout` seconds.

```python
driver.find(query="Submit").tap()
el = driver.wait_for_text("Order confirmed", timeout=30.0)
print(el.text)
```

### `wait_until_gone()`

```python
driver.wait_until_gone(
    text: str,
    *,
    timeout: float = 10.0,
    poll_ms: int = 300,
    exact: bool = False,
    ocr_engine: OcrEngine = "free",
) -> None
```

Polls until `text` disappears from the screen or `timeout` seconds elapse. Useful for waiting on loading spinners, progress bars, or transient messages.

<ParamField body="text" type="str" required>
  Text to wait for disappearance of.
</ParamField>

<ParamField body="timeout" type="float" default="10.0">
  Total seconds to wait before raising `TimeoutError`.
</ParamField>

<ParamField body="poll_ms" type="int" default="300">
  Milliseconds between polls.
</ParamField>

<ParamField body="exact" type="bool" default="False">
  Whether to require an exact match when checking for presence.
</ParamField>

<ParamField body="ocr_engine" type="OcrEngine" default='"free"'>
  OCR backend to use on each poll.
</ParamField>

**Returns** `None`.  
**Raises** `TimeoutError` if the text is still present after `timeout` seconds.

```python
driver.find(query="submit").tap()
driver.wait_until_gone("Loading...", timeout=20.0)
screen = driver.observe()
```

### `wait_for()`

```python
driver.wait_for(
    predicate: Callable[[Screen], object],
    *,
    timeout: float = 10.0,
    poll_ms: int = 300,
) -> Screen
```

Repeatedly calls `observe()` and passes the result to `predicate`. Returns the first `Screen` for which `predicate` returns a truthy value. Use this for complex multi-condition checks that don't map cleanly to a single text query.

<ParamField body="predicate" type="Callable[[Screen], object]" required>
  A callable that receives a `Screen` and returns truthy when the desired
  state is reached.
</ParamField>

<ParamField body="timeout" type="float" default="10.0">
  Total seconds to wait before raising `TimeoutError`.
</ParamField>

<ParamField body="poll_ms" type="int" default="300">
  Milliseconds between polls.
</ParamField>

**Returns** `Screen` — the snapshot that satisfied the predicate.  
**Raises** `TimeoutError` if the predicate is not satisfied within `timeout` seconds.

```python
# Wait until both "Username" and "Password" are visible
screen = driver.wait_for(
    lambda s: s.find_text("Username") and s.find_text("Password"),
    timeout=15.0,
)
```

## Interaction methods

### `tap()`

```python
driver.tap(coords: Coords) -> None
```

Sends a single tap gesture at the given screen coordinates. To tap an element, prefer `element.tap()` which resolves coordinates automatically.

<ParamField body="coords" type="Coords" required>
  A `{"x": int, "y": int}` dict in frame-space pixels.
</ParamField>

```python
driver.tap({"x": 540, "y": 960})
```

### `long_press()`

```python
driver.long_press(coords: Coords, *, duration_ms: int = 800) -> None
```

Sends a press-and-hold gesture at the given coordinates.

<ParamField body="coords" type="Coords" required>
  Tap target in frame-space pixels.
</ParamField>

<ParamField body="duration_ms" type="int" default="800">
  How long to hold the press, in milliseconds.
</ParamField>

```python
driver.long_press({"x": 200, "y": 400}, duration_ms=1500)
```

### `swipe()`

```python
driver.swipe(start: Coords, end: Coords, *, duration_ms: int = 300) -> None
```

Performs a swipe gesture from `start` to `end` over `duration_ms` milliseconds.

<ParamField body="start" type="Coords" required>
  Starting point of the swipe in frame-space pixels.
</ParamField>

<ParamField body="end" type="Coords" required>
  Ending point of the swipe in frame-space pixels.
</ParamField>

<ParamField body="duration_ms" type="int" default="300">
  Gesture duration in milliseconds. Longer durations produce slower, more
  deliberate swipes.
</ParamField>

```python
# Scroll down
driver.swipe({"x": 540, "y": 1200}, {"x": 540, "y": 400}, duration_ms=500)
```

### `type_text()`

```python
driver.type_text(text: str) -> None
```

Types a string using US-layout key events. The target element must already be focused. To focus and type in a single call, use `element.type_into()`.

<ParamField body="text" type="str" required>
  The string to type. Characters must be typable on a standard US keyboard
  layout.
</ParamField>

```python
driver.find(query="search box").tap()
driver.type_text("Axilio automation")
```

### `key_press()`

```python
driver.key_press(key: int) -> None
```

Emits a single HID consumer-page key press. Use the `Key` class constants for named keys.

<ParamField body="key" type="int" required>
  A HID Usage Table consumer-page code. Use constants from the `Key` class
  (e.g. `Key.HOME`, `Key.BACK`).
</ParamField>

```python
from axilio.drivers.mobile.keys import Key

driver.key_press(Key.HOME)
driver.key_press(Key.BACK)
driver.key_press(Key.VOLUME_UP)
```

### `screenshot()`

```python
driver.screenshot() -> bytes
```

Captures the current frame and returns it as PNG-encoded bytes. Useful for visual assertions, debugging artifacts, or attaching screenshots to test reports.

**Returns** `bytes` — a valid PNG image.  
**Raises** `InternalError` if the daemon returns no result or malformed data.

```python
png_bytes = driver.screenshot()
with open("screenshot.png", "wb") as f:
    f.write(png_bytes)
```

### `close()`

```python
driver.close() -> None
```

Releases the underlying transport. The next method call on the same driver instance will reconnect automatically. Use this to free the socket when you're done driving the device but still want to keep the allocation alive.

## Key constants

The `Key` class provides named constants for common HID consumer-page codes. Pass any constant to `driver.key_press()`.

| Constant | HID Code | Description |
|---|---|---|
| `Key.HOME` | `0x0223` | Navigate to the home screen |
| `Key.BACK` | `0x0224` | Navigate back |
| `Key.MENU` | `0x0040` | Open the app menu |
| `Key.RECENTS` | `0x0301` | Open recent apps |
| `Key.POWER` | `0x0030` | Toggle the power state |
| `Key.VOLUME_UP` | `0x00E9` | Increase volume |
| `Key.VOLUME_DOWN` | `0x00EA` | Decrease volume |
| `Key.MUTE` | `0x00E2` | Toggle mute |
| `Key.PLAY_PAUSE` | `0x00CD` | Play or pause media |
| `Key.NEXT_TRACK` | `0x00B5` | Skip to the next track |
| `Key.PREV_TRACK` | `0x00B6` | Skip to the previous track |
| `Key.STOP` | `0x00B7` | Stop media playback |
| `Key.SEARCH` | `0x0221` | Open the system search |
| `Key.ASSIST` | `0x01CB` | Invoke the device assistant |

```python
from axilio.drivers.mobile.keys import Key
from axilio.drivers import MobileDriver

driver = MobileDriver.connect()
driver.key_press(Key.HOME)       # Go home
driver.key_press(Key.RECENTS)    # Open recents
driver.key_press(Key.BACK)       # Navigate back
```

## Types reference

### `Element`

Returned by all selector methods. Fields are read-only. Actions chain directly off the object.

<ResponseField name="bbox" type="BBox">
  Bounding box of the element in frame-space pixels. See [`BBox`](#bbox).
</ResponseField>

<ResponseField name="center" type="Coords">
  Pre-computed center point derived from `bbox`. See [`Coords`](#coords).
</ResponseField>

<ResponseField name="confidence" type="float">
  Detection confidence in `[0.0, 1.0]`. OCR elements include word-level
  confidence; VLM elements reflect model certainty.
</ResponseField>

<ResponseField name="text" type="str | None">
  The OCR-recognised text for this element, or `None` for icon-only elements
  located by the VLM.
</ResponseField>

<ResponseField name="source" type="'ocr' | 'vlm'">
  How the element was located. `"ocr"` for text-based selectors; `"vlm"` for
  elements found purely via `find()` without OCR text.
</ResponseField>

#### Element action methods

| Method | Signature | Description |
|---|---|---|
| `tap()` | `-> None` | Tap at the element's center |
| `long_press()` | `(*, duration_ms: int = 800) -> None` | Press-and-hold at the element's center |
| `type_into()` | `(text: str) -> None` | Tap the element, then type `text` |
| `swipe_to()` | `(other: Element, *, duration_ms: int = 300) -> None` | Swipe from this element's center to `other`'s center |

```python
# Chain actions on an Element
driver.find(query="email field").type_into("user@example.com")
driver.find(query="next button").tap()

# Swipe from one element to another
start = driver.find_text("Start")
end   = driver.find_text("End")
start.swipe_to(end, duration_ms=500)
```

### `Screen`

An immutable snapshot of one observed frame. Returned by `observe()` and `wait_for()`.

<ResponseField name="texts" type="list[Element]">
  All OCR-detected text elements in the frame.
</ResponseField>

<ResponseField name="icons" type="list[IconBox]">
  All YOLO-detected icon bounding boxes in the frame.
</ResponseField>

<ResponseField name="hash" type="str">
  Perceptual hash of the frame. Compare hashes to detect whether the screen
  has changed between two observations.
</ResponseField>

<ResponseField name="width" type="int">
  Frame width in pixels.
</ResponseField>

<ResponseField name="height" type="int">
  Frame height in pixels.
</ResponseField>

<ResponseField name="captured_at" type="datetime">
  UTC timestamp at which the frame was captured.
</ResponseField>

`Screen` also exposes `find_text(text, *, exact=False)` and `find_all_text(*, contains=None, pattern=None)` — identical in behaviour to the driver methods of the same name but operating on already-captured data without an additional `observe()` round-trip.

### `IconBox`

A detected icon region. Unlike `Element`, an `IconBox` carries no text and no driver reference — it is purely geometric.

<ResponseField name="bbox" type="BBox">Bounding box in frame-space pixels.</ResponseField>
<ResponseField name="center" type="Coords">Pre-computed center of the bounding box.</ResponseField>
<ResponseField name="confidence" type="float">YOLO detection confidence in `[0.0, 1.0]`.</ResponseField>

### `BBox`

A `TypedDict` representing an axis-aligned rectangle with top-left origin.

| Key | Type | Description |
|---|---|---|
| `x` | `int` | Left edge in pixels |
| `y` | `int` | Top edge in pixels |
| `width` | `int` | Width in pixels |
| `height` | `int` | Height in pixels |

### `Coords`

A `TypedDict` representing a point in frame-space pixels.

| Key | Type | Description |
|---|---|---|
| `x` | `int` | Horizontal position |
| `y` | `int` | Vertical position |

## Exception reference

| Exception | Retryable | When raised |
|---|---|---|
| `ElementNotFoundError` | No | `find()` found no matching element |
| `TimeoutError` | Yes | A `wait_*` method or `find()` exceeded its deadline |
| `DeviceOfflineError` | Yes | Device is transiently unavailable |
| `NotConnectedError` | No | Daemon could not reach the on-device agent |
| `NoAllocationError` | No | Daemon has no active allocation |
| `UnauthorizedError` | No | Session token rejected by the agent |
| `InvalidArgsError` | No | Arguments failed validation |
| `InternalError` | No | Unclassified failure on the daemon side |
| `ConnectionError` | No | SDK could not open the Unix socket |

```python
from axilio.drivers.mobile._errors import ElementNotFoundError, TimeoutError

try:
    el = driver.find(query="confirm button", timeout=5.0)
    el.tap()
except ElementNotFoundError:
    print("Confirm button not found — taking a screenshot for debugging")
    png = driver.screenshot()
    with open("debug.png", "wb") as f:
        f.write(png)
except TimeoutError:
    print("Timed out waiting for VLM response")
```
