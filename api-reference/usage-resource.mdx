The `client.usage` resource gives you full visibility into how your organisation consumes Axilio's compute capacity. You can pull aggregated compute-minute totals bucketed by hour, day, or month for cost-forecasting and alerting, and you can drill into individual device sessions to audit exactly which workflows drove usage. Both methods work in local and sandbox modes, so scheduled scripts can track their own consumption in real time without any special permissions.

<Note>
  All timestamps returned by this resource are **timezone-aware UTC datetimes**. Always pass timezone-aware `datetime` objects as `start` and `end` to avoid ambiguous local-time comparisons.
</Note>

---

## `metrics(start, end, granularity)`

Returns aggregated compute-minute usage for your organisation across the half-open interval `[start, end)`. The result includes per-bucket breakdowns (one entry per hour, day, or calendar month, depending on `granularity`) and rolled-up totals for the entire window.

**Available in:** local mode · sandbox mode

<ParamField path="start" type="datetime" required>
  Start of the query window, **inclusive**. Must be a timezone-aware `datetime`. The backend aligns this to the nearest bucket boundary for the chosen granularity — for example, `"daily"` snaps to midnight UTC.
</ParamField>

<ParamField path="end" type="datetime" required>
  End of the query window, **exclusive**. Must be a timezone-aware `datetime` after `start`. Buckets that begin before `end` are included; any bucket that starts on or after `end` is excluded.
</ParamField>

<ParamField path="granularity" type='"hourly" | "daily" | "monthly"'>
  Time resolution for the bucket breakdown. Defaults to `"daily"`.

  - `"hourly"` — one `UsageBucket` per clock-hour; useful for debugging sudden spikes.
  - `"daily"` — one `UsageBucket` per calendar day; the default for dashboards and alerts.
  - `"monthly"` — one `UsageBucket` per calendar month; useful for billing reconciliation.
</ParamField>

<CodeGroup>
```python Daily usage for the past 30 days
from datetime import datetime, timedelta, timezone

now   = datetime.now(timezone.utc)
start = now - timedelta(days=30)

metrics = client.usage.metrics(start=start, end=now)

print(f"Total minutes: {metrics.total_minutes:.1f}")
print(f"Total cost:    ${metrics.total_cost_microdollars / 1_000_000:.4f}")

for bucket in metrics.buckets:
    print(bucket.bucket_start.date(), f"{bucket.minutes:.1f} min")
```

```python Hourly — last 24 hours
from datetime import datetime, timedelta, timezone

now   = datetime.now(timezone.utc)
start = now - timedelta(hours=24)

metrics = client.usage.metrics(
    start=start,
    end=now,
    granularity="hourly",
)

for bucket in metrics.buckets:
    print(f"{bucket.bucket_start:%H:%M}  {bucket.minutes:.2f} min")
```

```python Monthly — year to date
from datetime import datetime, timezone

start = datetime(2025, 1, 1, tzinfo=timezone.utc)
end   = datetime(2025, 8, 1, tzinfo=timezone.utc)

metrics = client.usage.metrics(
    start=start,
    end=end,
    granularity="monthly",
)

for bucket in metrics.buckets:
    print(f"{bucket.bucket_start:%Y-%m}  {bucket.minutes:.0f} min")
```
</CodeGroup>

### Returns — `UsageMetrics`

<ResponseField name="start" type="datetime" required>
  Effective start of the query window as interpreted by the backend (may be snapped to a bucket boundary).
</ResponseField>

<ResponseField name="end" type="datetime" required>
  Effective end of the query window as interpreted by the backend.
</ResponseField>

<ResponseField name="granularity" type="str" required>
  The granularity used for this response — echoes back the value you passed (or `"daily"` when you used the default).
</ResponseField>

<ResponseField name="total_minutes" type="float" required>
  Total compute-minutes consumed across the entire `[start, end)` window, summed across all buckets and all devices.
</ResponseField>

<ResponseField name="total_cost_microdollars" type="int" required>
  Total cost for the window in microdollars (divide by `1_000_000` for a dollar figure).
</ResponseField>

<ResponseField name="buckets" type="list[UsageBucket]" required>
  Ordered list of time-sliced usage buckets. The list is sorted by `bucket_start` ascending.

  <Expandable title="UsageBucket fields">
    <ResponseField name="bucket_start" type="datetime" required>
      UTC datetime marking the **start** of this bucket. For `"daily"` granularity this is always midnight UTC; for `"hourly"` it is the top of the hour; for `"monthly"` it is the first of the month at midnight UTC.
    </ResponseField>

    <ResponseField name="minutes" type="float" required>
      Compute-minutes consumed during this bucket across all devices in the org.
    </ResponseField>

    <ResponseField name="cost_microdollars" type="int" required>
      Cost for this bucket in microdollars, computed at the rate active for your plan during the bucket's time window.
    </ResponseField>
  </Expandable>
</ResponseField>

## `sessions(start, end, workflow_id)`

Returns the individual device sessions that occurred within `[start, end)`. Each session corresponds to one device allocation that was started and (if completed) ended within your org. You can optionally filter to sessions belonging to a single workflow.

**Available in:** local mode · sandbox mode

<Tip>
  Combine `sessions()` with `metrics()` for full cost attribution: use `metrics()` to find periods of unusually high usage, then call `sessions()` over the same window (filtered by `workflow_id`) to pinpoint which automation drove it.
</Tip>

<ParamField path="start" type="datetime" required>
  Start of the query window, **inclusive**, based on `started_at`. Must be a timezone-aware `datetime`.
</ParamField>

<ParamField path="end" type="datetime" required>
  End of the query window, **exclusive**, based on `started_at`. Sessions that began before `end` are included regardless of when they ended.
</ParamField>

<ParamField path="workflow_id" type="str | None">
  When provided, only sessions belonging to this workflow are returned. Omit (or pass `None`) to return sessions from all workflows in the org.
</ParamField>

<CodeGroup>
```python All sessions in the past 7 days
from datetime import datetime, timedelta, timezone

now   = datetime.now(timezone.utc)
start = now - timedelta(days=7)

sessions = client.usage.sessions(start=start, end=now)

for s in sessions:
    duration = s.minutes
    print(f"{s.id}  device={s.device_id}  {duration:.1f} min")
```

```python Filter by workflow
from datetime import datetime, timedelta, timezone

now   = datetime.now(timezone.utc)
start = now - timedelta(days=30)

sessions = client.usage.sessions(
    start=start,
    end=now,
    workflow_id="wf_01jx4abc...",
)

total = sum(s.minutes for s in sessions)
print(f"Workflow consumed {total:.1f} min across {len(sessions)} sessions")
```

```python Find sessions that are still running
from datetime import datetime, timedelta, timezone

now     = datetime.now(timezone.utc)
start   = now - timedelta(hours=2)

running = [
    s for s in client.usage.sessions(start=start, end=now)
    if s.ended_at is None
]
print(f"{len(running)} session(s) still active")
```
</CodeGroup>

### Returns — `list[Session]`

Each element is a `Session` object with the following fields:

<ResponseField name="id" type="str" required>
  Unique session identifier for this device allocation.
</ResponseField>

<ResponseField name="workflow_id" type="str | None" required>
  The workflow that created this session, or `None` for sessions started outside a named workflow (for example, interactive API calls).
</ResponseField>

<ResponseField name="run_id" type="str | None" required>
  The specific workflow run identifier, or `None` when not applicable. Useful for correlating sessions with individual run logs.
</ResponseField>

<ResponseField name="device_id" type="str" required>
  The identifier of the mobile device (emulated or physical) that was allocated for this session.
</ResponseField>

<ResponseField name="started_at" type="datetime" required>
  UTC datetime when the device was allocated and the session began.
</ResponseField>

<ResponseField name="ended_at" type="datetime | None" required>
  UTC datetime when the session ended and the device was released, or `None` if the session is still active at the time of the API call.
</ResponseField>

<ResponseField name="minutes" type="float" required>
  Compute-minutes consumed by this session. For active sessions this is the accrued usage up to the moment the API responded.
</ResponseField>

<ResponseField name="cost_microdollars" type="int" required>
  Cost for this session in microdollars, billed at the rate active for your plan at session start.
</ResponseField>

## Error Reference

| Exception | When it is raised |
|---|---|
| `UnauthorizedError` | API key is missing, malformed, or has been revoked. |
| `NotFoundError` | The specified `workflow_id` does not exist in the org. |
| `RateLimitError` | Too many requests in a short window — back off and retry. |
| `ServerError` | Transient backend error (5xx) — safe to retry with exponential backoff. |
