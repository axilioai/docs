Argus is Axilio's vision inference service. You send it a screenshot or any image and it returns structured data: YOLO-detected UI icons with bounding boxes, OCR-extracted text regions, and — through its VLM-backed locator — a pixel-precise answer to a natural-language query such as *"where is the sign-in button?"*. All three capabilities share a single base URL and the same API key you already use for the rest of the Axilio platform. You can call Argus directly from your own code, or let MobileDriver invoke it automatically during device-control sessions — the choice depends on your use case.

## Capabilities

Argus exposes three endpoints:

| Endpoint | What it does |
|---|---|
| `POST /api/v1/inference/infer` | Runs YOLO object detection, OCR, or both over an image. Returns icon detections, text regions, perceptual hashes, and timing metadata. |
| `POST /api/v1/inference/locate` | Asks a VLM to find the UI element described by a natural-language query. Accepts pre-computed OCR text as grounding context to improve accuracy. |
| `GET /api/v1/inference/models` | Lists the vision-language models available for `/locate`. No authentication required. |

## Relationship to MobileDriver

When you run automated sessions through `MobileDriver`, the SDK calls Argus internally on every frame — you never need to wire it up yourself. `client.argus` is the *direct* public surface for situations where you want to analyse screenshots yourself, build custom pipelines, pre-compute and cache OCR results, or integrate Argus into tooling that operates outside a live device session.

## Two Ways to Call Argus

<Tabs>
  <Tab title="Python SDK">
    The `client.argus` resource wraps the HTTP API and handles authentication, base64 encoding, and response mapping for you. It works in both local and sandbox modes.

    ```python
    import axilio

    client = axilio.Client()  # reads AXILIO_API_KEY from env

    with open("screenshot.png", "rb") as f:
        image_bytes = f.read()

    # Object detection + OCR in one call
    result = client.argus.infer(image_bytes, inference_type="combined")

    # Find a UI element by description
    location = client.argus.locate(image_bytes, query="the continue button")
    ```
  </Tab>
  <Tab title="HTTP API">
    You can call Argus from any HTTP client. Encode your image as standard base64 (no data-URI prefix) and include your API key in the `X-Axilio-Api-Key` header.

    ```bash
    curl -X POST https://api.axilio.ai/api/v1/inference/infer \
      -H "X-Axilio-Api-Key: axl_••••" \
      -H "Content-Type: application/json" \
      -d '{
        "image": "<base64-encoded-image>",
        "inference_type": "combined"
      }'
    ```
  </Tab>
</Tabs>

## Authentication

Every authenticated endpoint requires the `X-Axilio-Api-Key` header. Your key starts with `axl_` and is the same credential you use across the Axilio platform. The `GET /models` endpoint is public and requires no key.

```
X-Axilio-Api-Key: axl_••••••••••••••••
```

## Common Use Cases

- **Screenshot analysis** — run `/infer` over a captured frame to extract all visible text and icon positions without starting a device session.
- **Custom automation pipelines** — feed `/infer` output directly into your own click-targeting or assertion logic.
- **OCR caching** — use the `hash_result` field from `/infer` to detect duplicate frames and skip redundant OCR calls.
- **Grounded element location** — pre-compute OCR once with `/infer`, cache the results keyed by perceptual hash, then pass them to `/locate` on every query to avoid paying for OCR twice.
- **Model comparison** — call `/models` to list available VLMs and their per-token pricing, then pick the right cost/accuracy trade-off for your task.
