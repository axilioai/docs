The `/infer` endpoint is Argus's core analysis pass. You send it a base64-encoded PNG or JPEG and choose whether to run YOLO object detection (returning bounding boxes for every recognised UI icon), OCR (returning every readable text region with its position and confidence score), or both in a single round-trip. The response also includes perceptual hashes of the image's content area, which you can use to detect duplicate frames and skip redundant inference. All results come back with timing and dimension metadata so you can profile your pipeline end-to-end.

## Request

**`POST https://api.axilio.ai/api/v1/inference/infer`**

**Authentication:** `X-Axilio-Api-Key` header required.

### Parameters

<ParamField body="image" type="string" required>
  Base64-encoded image data (PNG or JPEG). Do not include a data-URI prefix — send the raw base64 string only.
</ParamField>

<ParamField body="inference_type" type="string" default="combined">
  Which analysis passes to run:

  - `"yolo"` — icon/object detection only. Skips OCR; `ocr_results` will be an empty array.
  - `"ocr"` — text recognition only. Skips YOLO; `icon_detections` will be an empty array.
  - `"combined"` — runs both passes in a single call. Use this when you need both detections and text regions, or when you want to feed the OCR output into a subsequent `/locate` call.
</ParamField>

<ParamField body="confidence_threshold" type="float" default="0.25">
  Minimum confidence score [0, 1] for a detection or OCR result to be included in the response. Raise this value to reduce noise; lower it to surface low-confidence matches. Applies to both YOLO detections and OCR results.
</ParamField>

<ParamField body="nms_iou_threshold" type="float" default="0.45">
  IoU (Intersection over Union) threshold for Non-Maximum Suppression, in the range [0, 1]. When two overlapping bounding boxes have an IoU above this value, only the higher-confidence box is kept. Lower values suppress more aggressively; higher values allow more overlap before suppression. Only affects icon detections — OCR is not filtered by NMS.
</ParamField>

<ParamField body="ocr_engine" type="string" default="free">
  Which OCR backend to use:

  - `"free"` — uses RapidOCR. Fast, no extra per-call cost, suitable for most UI text.
  - `"premium"` — uses a higher-accuracy engine billed per page. The underlying provider is abstracted; you will not see provider names in the response.

  Has no effect when `inference_type` is `"yolo"`.
</ParamField>

## Response

The response object wraps all results under a `data` field alongside envelope metadata.

<ResponseField name="type" type="string">
  Response envelope type. Value is always `"analyzer_result"`.
</ResponseField>

<ResponseField name="analyzer_type" type="string">
  Which analyzer was used. Value is always `"all"`.
</ResponseField>

<ResponseField name="data" type="object">
  The inference payload.

  <Expandable title="data fields">
    <ResponseField name="icon_detections" type="Detection[]">
      YOLO detections after NMS filtering. Empty when `inference_type` is `"ocr"`.

      <Expandable title="Detection fields">
        <ResponseField name="bbox" type="BoundingBox">
          Axis-aligned bounding box in image-pixel coordinates.

          <Expandable title="BoundingBox fields">
            <ResponseField name="x1" type="float">Top-left x coordinate, in pixels.</ResponseField>
            <ResponseField name="y1" type="float">Top-left y coordinate, in pixels.</ResponseField>
            <ResponseField name="x2" type="float">Bottom-right x coordinate, in pixels.</ResponseField>
            <ResponseField name="y2" type="float">Bottom-right y coordinate, in pixels.</ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="confidence" type="float">Detection confidence score in [0, 1].</ResponseField>
        <ResponseField name="class_id" type="integer">Numeric class identifier for the detected object type.</ResponseField>
        <ResponseField name="class_name" type="string">Human-readable label for the detected object type, e.g. `"button"` or `"icon"`.</ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="ocr_results" type="OCRResult[]">
      Text regions recognised by the OCR pass. Empty when `inference_type` is `"yolo"`.

      <Expandable title="OCRResult fields">
        <ResponseField name="text" type="string">The recognised text string.</ResponseField>
        <ResponseField name="confidence" type="float">OCR confidence score in [0, 1].</ResponseField>
        <ResponseField name="bbox" type="BoundingBox">
          Position of the text region in image-pixel coordinates (same `x1, y1, x2, y2` schema as icon detections).
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="hash_result" type="HashResult | null">
      Perceptual hashes of the image's content area. `null` when the YOLO pass is skipped.

      <Expandable title="HashResult fields">
        <ResponseField name="phash" type="string">Perceptual hash (DCT-based). Most robust to minor rendering differences.</ResponseField>
        <ResponseField name="ahash" type="string">Average hash. Fastest to compute; sensitive to brightness shifts.</ResponseField>
        <ResponseField name="dhash" type="string">Difference hash. Good at detecting structural changes between frames.</ResponseField>
        <ResponseField name="content_bounds" type="ContentBounds | null">
          The vertical slice of the image used for hashing — excludes notification bar and navigation bar.

          <Expandable title="ContentBounds fields">
            <ResponseField name="top_y" type="integer">Top Y coordinate of the content area (below the notification bar).</ResponseField>
            <ResponseField name="bottom_y" type="integer">Bottom Y coordinate of the content area (above the navigation bar).</ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="histogram" type="integer[][] | null">
          RGB colour histogram: 3 channels × 32 bins each. Useful for detecting colour-palette changes between frames.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="metadata" type="InferenceMetadata">
      Timing and image dimension information.

      <Expandable title="InferenceMetadata fields">
        <ResponseField name="inference_time_ms" type="float">Total server-side inference time in milliseconds.</ResponseField>
        <ResponseField name="image_width" type="integer">Width of the submitted image in pixels.</ResponseField>
        <ResponseField name="image_height" type="integer">Height of the submitted image in pixels.</ResponseField>
        <ResponseField name="icon_detections_before_nms" type="integer">Raw YOLO detection count before Non-Maximum Suppression filtering.</ResponseField>
        <ResponseField name="icon_detections_after_nms" type="integer">Detection count after NMS — the length of `icon_detections` in the response.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Examples

<CodeGroup>

```python SDK
import axilio

client = axilio.Client()  # reads AXILIO_API_KEY from env

with open("screenshot.png", "rb") as f:
    image_bytes = f.read()

result = client.argus.infer(
    image_bytes,
    inference_type="combined",
    confidence_threshold=0.3,
    ocr_engine="free",
)

# Print all detected text
for ocr in result.ocr_results:
    print(f"{ocr.text!r}  conf={ocr.confidence:.2f}  bbox={ocr.bbox}")

# Print all detected icons
for icon in result.icons:
    print(f"{icon.class_name}  conf={icon.confidence:.2f}  bbox={icon.bbox}")

print(f"Inference took {result.inference_time_ms:.1f} ms")
```

```python OCR only
result = client.argus.infer(
    image_bytes,
    inference_type="ocr",
    ocr_engine="premium",   # higher accuracy, billed per page
    confidence_threshold=0.5,
)

for ocr in result.ocr_results:
    print(ocr.text, ocr.bbox)
```

## Non-Maximum Suppression

YOLO often produces multiple overlapping bounding boxes for a single object. Non-Maximum Suppression (NMS) removes the redundant ones: after ranking all detections by confidence, it suppresses any box whose IoU with a higher-confidence neighbour exceeds `nms_iou_threshold`. You can observe the effect by comparing `metadata.icon_detections_before_nms` with `metadata.icon_detections_after_nms`. Lowering `nms_iou_threshold` is more aggressive — it removes boxes that overlap only loosely — while a higher value tolerates more overlap before suppression.

## Free vs. Premium OCR

<Tabs>
  <Tab title="Free engine">
    The free engine (RapidOCR) runs entirely on Argus infrastructure with no additional per-call cost. It handles clear UI text well, especially Latin-script content with standard fonts. Use it for high-frequency inference loops where per-call billing would accumulate quickly.
  </Tab>
  <Tab title="Premium engine">
    The premium engine routes through a high-accuracy provider and is billed per page. It handles low-contrast text, stylised fonts, non-Latin scripts, and dense layouts better than the free engine. The provider name is never surfaced in the response. Use it when OCR accuracy directly affects downstream automation correctness.
  </Tab>
</Tabs>

## Using Hash Results for Frame Deduplication

The `hash_result` field in the HTTP response lets you avoid re-running inference on frames that haven't changed. The perceptual hash is part of the raw JSON response body and is not exposed on the `InferenceResult` SDK object — use the HTTP API directly when you want to implement hash-based caching:

```python
import base64
import httpx

cache: dict[str, dict] = {}

def analyse_http(api_key: str, image_bytes: bytes) -> dict:
    """Call /infer over HTTP and cache by perceptual hash."""
    payload = {
        "image": base64.b64encode(image_bytes).decode(),
        "inference_type": "combined",
    }
    resp = httpx.post(
        "https://api.axilio.ai/api/v1/inference/infer",
        headers={"X-Axilio-Api-Key": api_key},
        json=payload,
    )
    resp.raise_for_status()
    data = resp.json()["data"]

    hash_result = data.get("hash_result")
    phash = hash_result["phash"] if hash_result else None

    if phash and phash in cache:
        return cache[phash]   # identical frame — skip billing

    if phash:
        cache[phash] = data
    return data
```

<Tip>
  The `content_bounds` inside `hash_result` tells you exactly which vertical slice of the image was hashed — the area between the notification bar and the navigation bar. Argus excludes those chrome regions so that a status-bar clock update does not invalidate an otherwise-identical frame.
</Tip>

## Feeding Infer Results into Locate

When you need to find a specific element after analysing a frame, pass the `ocr_results` from `/infer` directly to `/locate` as grounding context. This avoids re-running OCR inside the VLM call and reduces both latency and cost:

```python
from axilio.argus import BoundingBox, TextElement

infer_result = client.argus.infer(image_bytes, inference_type="combined")

w = infer_result.image_width
h = infer_result.image_height

# TextElement.bbox must be normalised to [0, 1]; OcrResult.bbox is in pixels.
texts = [
    TextElement(
        text=r.text,
        bbox=BoundingBox(
            x1=r.bbox.x1 / w,
            y1=r.bbox.y1 / h,
            x2=r.bbox.x2 / w,
            y2=r.bbox.y2 / h,
        ),
    )
    for r in infer_result.ocr_results
]

location = client.argus.locate(
    image_bytes,
    query="the forgot password link",
    texts=texts,
)
```

<Note>
  `/locate` does **not** re-run OCR internally. If you omit `texts`, the VLM reasons from the image pixels alone, which is slower and typically less accurate for text-heavy interfaces. Passing pre-computed OCR is always recommended.
</Note>
