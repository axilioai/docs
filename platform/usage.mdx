Axilio records every second your automations consume phone time and exposes that data through two complementary APIs: `client.usage.metrics()` for time-bucketed aggregates and `client.usage.sessions()` for individual device-session records. Both APIs are available in local **and** sandbox mode, so your workflows can inspect their own consumption at runtime. Usage data maps directly to the line items on your invoices — the same underlying records that Axilio bills from are the ones you query here.

<Note>
  Always pass **timezone-aware** `datetime` objects to both `metrics()` and `sessions()`. The SDK passes timestamps directly to the backend, which stores everything in UTC. Naive datetimes (no `tzinfo`) will raise a `ValueError` in a future SDK version and may produce incorrect results today.
</Note>

## Aggregated metrics

`client.usage.metrics(start, end, granularity=...)` returns a single `UsageMetrics` object containing time-bucketed usage across your chosen window. Use this endpoint to draw cost graphs, build budget alerts, or check whether yesterday's batch run stayed within expected bounds.

### Granularity options

| Value | Bucket width | Best for |
|---|---|---|
| `"hourly"` | 1 hour | Intraday cost spikes, debugging a specific run |
| `"daily"` | 1 calendar day | Weekly spend reviews, anomaly detection |
| `"monthly"` | 1 calendar month | Billing reconciliation, year-over-year trends |

### Example: daily usage for the past 7 days

```python
from datetime import datetime, timedelta, timezone
import axilio

client = axilio.Client()

end = datetime.now(tz=timezone.utc)
start = end - timedelta(days=7)

metrics = client.usage.metrics(start, end, granularity="daily")

print(f"Period: {metrics.start.date()} → {metrics.end.date()}")
print(f"Total compute time: {metrics.total_minutes:.1f} minutes")
print(f"Total cost: ${metrics.total_cost_microdollars / 1_000_000:.4f}")
print()

for bucket in metrics.buckets:
    cost = bucket.cost_microdollars / 1_000_000
    print(f"  {bucket.bucket_start.date()}  {bucket.minutes:6.1f} min  ${cost:.4f}")
```

### UsageMetrics fields

<ResponseField name="metrics.start" type="datetime">
  The inclusive start of the query window, echoed back from your request.
</ResponseField>
<ResponseField name="metrics.end" type="datetime">
  The exclusive end of the query window, echoed back from your request.
</ResponseField>
<ResponseField name="metrics.granularity" type="Literal['hourly', 'daily', 'monthly']">
  The granularity you requested. Bucket widths in `buckets` match this value.
</ResponseField>
<ResponseField name="metrics.buckets" type="list[UsageBucket]">
  Ordered list of time-bucketed slices. Each bucket covers exactly one granularity period. Buckets with zero usage are included so you can render a complete time series without gap-filling.
</ResponseField>
<ResponseField name="metrics.total_minutes" type="float">
  Sum of `minutes` across all buckets in the window. Equivalent to summing `[b.minutes for b in metrics.buckets]`.
</ResponseField>
<ResponseField name="metrics.total_cost_microdollars" type="int">
  Total cost for the window in millionths of a US dollar. Divide by `1_000_000` for a dollar amount.
</ResponseField>

### UsageBucket fields

<ResponseField name="bucket.bucket_start" type="datetime">
  UTC start of this bucket. For daily granularity this is midnight UTC; for hourly it is the top of the hour.
</ResponseField>
<ResponseField name="bucket.minutes" type="float">
  Compute minutes consumed within this bucket across all sessions and workflows.
</ResponseField>
<ResponseField name="bucket.cost_microdollars" type="int">
  Cost incurred within this bucket, in millionths of a US dollar.
</ResponseField>

## Per-session records

`client.usage.sessions(start, end, workflow_id=None)` returns a list of individual `Session` objects — one per device allocation lifecycle. Use sessions to audit exactly which workflow runs drove cost, attribute spend to specific automations, or investigate a spike identified in the metrics view.

### Example: all sessions in a window

```python
from datetime import datetime, timedelta, timezone
import axilio

client = axilio.Client()

end = datetime.now(tz=timezone.utc)
start = end - timedelta(days=1)

sessions = client.usage.sessions(start, end)

for s in sessions:
    duration = (s.ended_at - s.started_at).total_seconds() / 60 if s.ended_at else None
    cost = s.cost_microdollars / 1_000_000
    print(f"  Session {s.id[:8]}  {s.minutes:5.1f} min  ${cost:.4f}  wf={s.workflow_id}")
```

### Example: filter sessions by workflow

Pass `workflow_id` to scope the results to a single workflow. This is the most efficient way to attribute cost to a specific automation without downloading all sessions and filtering client-side.

```python
SCRAPER_WORKFLOW_ID = "wf_01j9abc123def456"

sessions = client.usage.sessions(
    start=datetime(2025, 7, 1, tzinfo=timezone.utc),
    end=datetime(2025, 8, 1, tzinfo=timezone.utc),
    workflow_id=SCRAPER_WORKFLOW_ID,
)

total_minutes = sum(s.minutes for s in sessions)
total_cost = sum(s.cost_microdollars for s in sessions) / 1_000_000

print(f"Workflow consumed {total_minutes:.1f} min costing ${total_cost:.4f} in July")
```

### Session fields

<ResponseField name="session.id" type="str">
  Unique session identifier. Use this to correlate with run logs or to cross-reference an invoice line item.
</ResponseField>
<ResponseField name="session.workflow_id" type="str | None">
  The workflow that triggered this session, if any. `None` for sessions started via direct `devices.allocate()` calls outside a workflow context.
</ResponseField>
<ResponseField name="session.run_id" type="str | None">
  The specific workflow run that triggered this session. Useful for tracing a single execution end-to-end.
</ResponseField>
<ResponseField name="session.device_id" type="str">
  The physical device that was allocated. You can use this with `client.devices` to look up device metadata.
</ResponseField>
<ResponseField name="session.started_at" type="datetime">
  UTC timestamp when the device was allocated and billing began.
</ResponseField>
<ResponseField name="session.ended_at" type="datetime | None">
  UTC timestamp when the device was released and billing stopped. `None` if the session is still active at query time.
</ResponseField>
<ResponseField name="session.minutes" type="float">
  Total billable minutes for this session, computed as `(ended_at - started_at)` in decimal minutes. For open sessions, this reflects accrued time as of the query.
</ResponseField>
<ResponseField name="session.cost_microdollars" type="int">
  Total cost for this session in millionths of a US dollar.
</ResponseField>

## Connecting usage to invoices

Usage data and invoices share the same underlying session records. If your invoice shows a charge you want to investigate, use `client.billing.invoices()` to get the invoice's `created_at` timestamp, then query `client.usage.sessions()` for the same period to see the individual sessions that composed it.

```python
from datetime import datetime, timezone

# 1. Find last month's invoice
invoices = client.billing.invoices(
    start=datetime(2025, 6, 1, tzinfo=timezone.utc),
    end=datetime(2025, 7, 1, tzinfo=timezone.utc),
)
inv = invoices[0]
print(f"Invoice {inv.id}: ${inv.amount_microdollars / 1_000_000:.2f}")

# 2. Pull sessions for the same billing window
sub = client.billing.subscription()
sessions = client.usage.sessions(
    start=sub.current_period_start,
    end=sub.current_period_end,
)

# 3. Find the costliest session
costliest = max(sessions, key=lambda s: s.cost_microdollars)
print(f"Most expensive session: {costliest.id} — {costliest.minutes:.1f} min")
```

<Tip>
  Build a lightweight cost-guard into long-running workflows by querying `client.usage.metrics()` with `granularity="hourly"` and comparing the last bucket's cost against a threshold. Raise an alert or halt the run before charges accumulate beyond budget.
</Tip>
