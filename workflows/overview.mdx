A **workflow** is your automation saved on Axilio: the code that drives a phone,
the same code you've seen in these docs. A **run** is one execution of a workflow.

Save a workflow once, then run it again and again, on demand or on a schedule.

## How it relates to sessions

When you write code with `session()`, you're driving a phone live. A workflow is
that same code, saved so Axilio can run it for you on phones it manages. The script
you test runs unchanged as a workflow.
